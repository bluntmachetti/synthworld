# Security policy

SynthWorld accepts only unmistakably synthetic identity data. If you discover
real personal data, a secret, or a way the generator can emit an identifier that
could be mistaken for a real person's, treat it as a security and safety issue.

Do not include sensitive details in a public issue. Use GitHub's private
vulnerability-reporting form in the repository's **Security** tab. For ordinary
non-sensitive defects, open a public issue with the smallest synthetic
reproduction possible.
